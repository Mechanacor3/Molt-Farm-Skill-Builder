# Run Summarizer

Use this skill when:
- A run has completed and needs a short summary.
- You need to capture what was attempted, what happened, and whether it succeeded.
- The output should be suitable for logs and human review.

Instructions:
1. Read the run record and any immediately relevant output summary.
2. State what was attempted, what happened, whether it succeeded, and what was produced.
3. Keep the result concise, structured, and factual.
4. Do not expand into unrelated analysis.
5. When useful, follow the output template in `@./references/run-summary-template.md`.
