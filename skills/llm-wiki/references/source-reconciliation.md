# Source Reconciliation

Treat each source type differently:

- Raw notes:
  - strongest evidence for what happened at the table or in the original capture
  - may contain shorthand, spelling drift, or incomplete context
- Reference text:
  - useful for background, chapter framing, or canonical naming
  - should not overwrite a table event that the notes clearly record
- Errata:
  - useful for corrected names, terms, or mechanics
  - apply corrections carefully and call out conflicts when the raw note uses an older form

When sources disagree:

1. Preserve the note-backed event.
2. Apply the corrected spelling or canonical label on index or entity pages when the correction is well supported.
3. Do not silently rewrite quoted or paraphrased raw-note evidence.
4. Mark the discrepancy as `variant in notes`, `conflicting`, or `unverified` when needed.

Good wiki behavior:
- "The raw note names `Zeiraun`; the canonical page and index use `Zelroun` per the reference and errata."
- "The note supports the meeting; the reference only supplies the broader chapter context."

Bad wiki behavior:
- inventing a clean resolution without naming the conflict
- creating a second page for a spelling variant
- replacing note-backed history with unsupported book detail
