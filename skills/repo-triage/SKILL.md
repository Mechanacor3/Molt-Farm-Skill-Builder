# Repo Triage

Use this skill when:
- A workflow needs a quick repository assessment.
- The user wants a short summary of what to inspect next.
- Inputs are narrow and local to the current repository.

Instructions:
1. Read only the task input and the minimal repository context needed.
2. Summarize the likely goal, current state, and next action.
3. Keep the output short and operational.
4. Do not behave like a chat assistant.
5. Use this checklist when helpful: @./references/triage-checklist.md
