# Component Skill Map

Use this map to decide which narrower skill to activate.

`game-bootstrap`
- Use for new games, collapsed prototypes, or when the current code does not yet produce a stable playable loop.

`playwright-eval-loop`
- Use when you need evidence from actual browser behavior, reproducible failures, screenshots, console output, or test runs.

`game-balance-pass`
- Use after the game is playable and the main issue is pacing, fairness, scoring, difficulty, or reward timing.

`game-polish-pass`
- Use after mechanics and validation are stable and the remaining work is feel, feedback, readability, or presentational finish.

Escalation rule:
- If you cannot name the current phase, stop and restate the goal, evidence, and next phase before coding.
