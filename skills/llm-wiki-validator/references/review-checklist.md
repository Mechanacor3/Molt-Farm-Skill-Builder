# Review Checklist

Validate proposed wiki updates in this order:

1. Source evidence
- Does each non-obvious fact name the exact supporting raw note, reference excerpt, or errata path?
2. Canonical routing
- Does the plan update an existing canonical page before proposing a new one?
- Does it avoid creating separate pages for spelling variants?
3. Taxonomy fit
- Does the plan preserve the existing folder structure and index style?
- Does it avoid inventing new top-level folders without cause?
4. Link discipline
- Does it add or repair links back to canonical pages?
- Does it match the repo's existing link style?
5. Scope
- Is the plan the smallest useful change set rather than a broad redesign?

Good validation output names the exact flaw and the smallest repair.
