# Skill Refiner

Use this skill when:
- A skill needs clearer instructions or narrower scope.
- Lessons from runs should be folded back into `SKILL.md`.
- You need to revise a skill without changing the overall architecture.

Instructions:
1. Read the target skill first, then any directly relevant lesson or log context.
2. Preserve the skill's intent and keep the revision minimal.
3. Prefer clearer triggers, tighter steps, and fewer ambiguous instructions.
4. Keep the result portable, human-readable, and local-first.
5. Do not add runtime concerns, UI concepts, or broad new behaviors.
6. Use `@./references/refinement-checklist.md` to keep revisions narrow and reviewable.
