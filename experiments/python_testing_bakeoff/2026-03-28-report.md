## What Was Done

### Fetch and staging

The following terminal tools were used:
- `git`
- `cp`
- `find`
- `sed`
- `rg`
- `python3`
- `./molt`

The upstream skill folders were fetched with sparse checkouts into `tmp/upstream_fetch/`, then copied into:
- `example_upstream_skills/python_testing_bakeoff/` as untouched source storage
- `skills/upstream-python-testing-bakeoff/` as local eval copies

The local eval copies intentionally changed only the skill `name:` metadata so all four skills could be discovered at the same time by `molt skill-builder eval-skill`.

### Shared eval suite

One shared six-case pytest suite was installed into each local eval copy under `evals/evals.json`.

The suite tested:
- TDD red-green-refactor guidance
- table-driven parametrization
- shared fixtures and `conftest.py`
- `monkeypatch` plus `tmp_path`
- async pytest patterns
- coverage guidance

## Sameness Detection Result

Output:

- The near-dupe scan found `2` candidate pairs out of the `6` possible pairings across the four imported skills.
- High severity pair:
  - `ecc-python-testing` vs `mindrally-python-testing`
  - score: `0.85`
  - reason: same skill name plus overlapping testing vocabulary
- Medium severity pair:
  - `ecc-python-testing` vs `wshobson-python-testing-patterns`
  - score: `0.514`
- No Luxor pair cleared the current `0.50` threshold.

Interpretation:
- The current scanner can detect obvious sameness.
- It does not fully surface all intuitive pytest-skill overlap in this set.
- For this bake-off, it partially detected sameness rather than exhaustively clustering the whole family.

## Reliability Caveat

The current evaluator has a grading fragility:
- `packages/moltfarm/skill_evaluator.py` aligns grader outputs back to checks by exact check text.
- When the grader does not echo a check string exactly, the check falls back to a false result with evidence text `The grader did not return a result for this check.`

Fallback-check counts observed in this bake-off:

| Skill | With Skill Fallback Checks | Baseline Fallback Checks | Total Checks Per Side |
| --- | ---: | ---: | ---: |
| `luxor-pytest-patterns` | `3` | `9` | `18` |
| `mindrally-python-testing` | `4` | `4` | `18` |
| `ecc-python-testing` | `8` | `7` | `18` |
| `wshobson-python-testing-patterns` | `12` | `7` | `18` |

Interpretation:
- The broad ordering from this run is still useful.
- Fine-grained differences are noisy until the grading alignment becomes less brittle.
- Luxor's win is the clearest signal.
- Mindrally's slight edge over baseline looks real enough to keep, but it is not a large margin.
- ECC and Wshobson underperformed here, but their exact deficit is likely overstated by evaluator brittleness.
