# Fishbowl Journal

Use this skill when:
- A fishbowl pass needs to be recorded.
- A backlog item, decision, or lesson candidate needs a small update.
- Evidence should be preserved without promoting a full repo lesson yet.

Instructions:
1. Read only the smallest relevant journal files first.
2. Use `journal/templates/session-template.md` for session notes.
3. Prefer updating one existing file over creating parallel notes.
4. Name exact evidence paths for every non-obvious claim.
5. If evidence is thin, write `TODO` or `unverified` instead of guessing.
6. Keep entries short and operational.
7. Report in exactly this order:
   entry_path:
   updated:
   evidence_paths:
   next:
